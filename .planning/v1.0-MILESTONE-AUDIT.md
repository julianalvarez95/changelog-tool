### Gap 2: `failed_repos` Deserialization Bug (Functional)

**Affects:** GENR-05 (partial-job progress), HIST-02 (partial-job history detail)

`web/db.py` stores `failed_repos` as a JSON string via `json.dumps()`. Both `get_job()` and `list_jobs()` return it as a raw string. The route handlers pass it to the API response without calling `json.loads()`. The UI receives a string and calls `.forEach()` on it — **TypeError at runtime** for any job with `status = "partial"`.

**Scope:** Only affects partial-status jobs (at least one repo failed). Jobs completing `done` or `failed` are unaffected.

**Fix:**
```python
# In web/db.py, at the end of get_job() and list_jobs():
if row_dict.get("failed_repos"):
    row_dict["failed_repos"] = json.loads(row_dict["failed_repos"])
else:
    row_dict["failed_repos"] = []
```
